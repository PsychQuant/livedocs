# LiveDocs — Positioning

> External-messaging source of truth. Companion to
> [`docs/design/primary-source-spectrum.md`](design/primary-source-spectrum.md)
> (the product boundary).

## One-liner

> **Live docs, matched to your installed version — straight from the source, on
> your machine. No stale index, no API key.**

## The wedge (what we lead with)

- **Headline (the pain):** always-latest, **straight from the source**, with no
  periodically-recrawled index sitting in between.
- **Reason-to-believe (the moat context7 can't copy):** it runs **locally and goes
  direct to the source** — which is *also* why it's private and needs no API key.
  context7 is architecturally a centralized service; it structurally cannot become
  local/private/keyless without abandoning its model.

Lead with the **outcome** (latest / private / keyless). "Decentralized" is the
mechanism that makes the outcome credible and uncopyable — it is the moat, not the
marquee.

## The sharpest angle: "the docs your code actually runs on"

Beyond "latest published," LiveDocs reconciles against your **locally installed
version** (version reconciliation + `introspect`). It answers with the docs for the
version *you actually have*, not just whatever is newest. This kills the version-skew
bug ("the docs say use X, but my installed version doesn't have it") — a dimension
even latest-only tools lack. This is the strongest differentiator; make it a lead,
not a footnote.

## Three differentiators (all flow from local + direct)

1. **Latest** — live from registry / `llms.txt` / repo / introspection; never a
   recrawled snapshot. *Proof:* context7 listed fastapi newest `0.128.0` while live
   PyPI was `0.138.2` (~10 versions behind).
2. **Private + zero-key** — queries never leave your machine; no account, no API key,
   no hosted rate-limit/quota.
3. **Raw + installed** — returns **verbatim** primary text AND can read your **actual
   installed** version/CLI. context7 returns ranked, LLM-summarized snippets only
   (generated by a model, often weeks old).

## Head-to-head vs context7

| Dimension | LiveDocs | context7 |
|---|---|---|
| Freshness | live, direct-to-source, ETag-revalidated | periodically recrawled → lags |
| Fidelity | raw verbatim reachable | ranked LLM-summarized snippets |
| Your environment | matched to installed version (reconciliation) | version-agnostic snapshot |
| Architecture | local tool, no backend | centralized hosted service |
| Privacy | queries stay on your machine | queries hit their servers |
| Access | no account, no API key | hosted (quota/keys) |
| Breadth | ~any public lib via generic discovery + adapters | large pre-indexed corpus |

## Honest reality: positioning ≠ winning

context7 leads on **distribution**, not fidelity — "better-but-niche loses to
worse-but-universal." A sharp message buys mindshare, not users. To actually compete
we still need context7's winning half:

- **Frictionless install** — already true (MCP + `docs-router` skill, one-line add).
- **Presence where devs look** — README, awesome-lists, being carried by default in
  other tools/plugins.
- **Breadth for the long tail** — via generic discovery (`llms.txt`/registry/repo) +
  a **shared adapter** marketplace (metadata, not content) — NOT a central cache.

## Wording guidance

- **Don't** lead with "decentralized" (abstract + buzzword-adjacent). Use concrete,
  unloaded phrasing: **"local / on your machine / straight from the source / no index
  in between."** Keep "decentralized" for the architecture explanation only.

## Candidate taglines

- **"The docs your code actually runs on."**
- **"Latest docs, straight from the source — not a stale index."**
- **"Live docs. On your machine. No index, no API key, no lag."**

## Anti-positioning (what NOT to build)

- A **central shared content cache** = becoming context7 (stale + crawl infra +
  privacy loss). Speed comes from **local ETag revalidation**; community effect comes
  from **shared routes/adapters**, never shared content. See the boundary in
  [`docs/design/primary-source-spectrum.md`](design/primary-source-spectrum.md).
